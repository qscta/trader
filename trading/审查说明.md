# 代码审查说明（给 Codex）

## 背景
这是一套**真实资金**的程序化合约交易系统，原为币安版（海龟通道突破 + 双均线 EMA 策略，已实盘跑通）。
本次改动把它**收敛为「欧易（OKX）单所版」**：只对接欧易，删除币安与多交易所架构。

## 核心原则（本次改动的边界）
> 只改三类东西：①统计展示口径 ②交易所范围（删币安、单所化）③默认参数（双均线短周期 6）。
> **不改**已跑通的交易逻辑：交易信号、下单、止损创建/推进、平仓、以损定量的仓位计算、调度执行。
> 适配欧易若需改接口调用，**只能在交易所适配层（okx_api.py / exchange_base.py）做映射**，不得改变策略层的输入输出语义。

## 请重点核对的 5 点
1. **交易逻辑等价性（最重要）**：`main.py` 里的交易信号、以损定量、开仓后建止损、止损推进、平仓、平仓后状态记录、每日 08:00 调度，是否与币安原版**行为等价**，确认只是替换了适配层，没有动策略语义。
2. **OKX 适配正确性（真钱红线）**：`okx_api.py` 的——
   - 合约**张数 ↔ 币数量**换算（每张 = `contractSize` 个币），是否在下单边界内部正确换算、不外泄；
   - 止损算法单 `stopLossPrice` + `reduceOnly` 是否正确（reduce-only、触发市价平仓）；
   - `cancel_all_orders` 能否连算法止损一起撤掉；
   - `set_leverage` / 单向（净）持仓模式 / `tdMode` 是否正确。
3. **状态隔离与迁移**：从多所版的 `data/okx/*` **迁回项目根目录**的逻辑（`main.py` 的 `_migrate_okx_legacy_state`）是否安全——根目录已有状态时不得覆盖，避免串仓/丢仓。
4. **未创新高统计口径**：`equity_tracker.py` 的 `build_account_stats()`——
   - `days_since_peak`（未创新高天数）、`longest_drawdown_days`（历史最长未创新高）；
   - 当前正处于未创新高时，历史最长须 `max(历史已记录, 当前进行中天数)`；
   - 创新高当天 days_since_peak 归零，并用「旧峰值时间→本次新高」结算刚结束的那段更新历史最长；
   - 前端 / `/api/account_stats` / 钉钉周报是否**统一复用**这一份结果（周报不得自己手算）。
5. **单所收敛是否彻底**：是否还有币安 / 多交易所残留（`exchange` 参数、`/api/overview` 等多所路由、前端交易所切换/汇总页）。

## 全量自审一轮（Fable 5）：新增修复清单

本轮按"只动适配层与明确 bug，不碰策略语义"的红线，从头通读全部源码后修复：

1. **OkxApi.get_position 归一化（高）**：OKX 可能返回 contracts=None/0 的空仓条目；原样外泄会让上层
   （币安时代写下的）`contracts == 0` 判断因 None 失效，盘中巡检 `contracts > 0` 更会直接 TypeError
   （每 5 分钟中断一次全品种巡检）。现无实仓统一返回 None（符合基类契约）。
2. **_fetch_algo_orders 不再吞异常（中）**：原先三种参数变体全失败时静默返回 []，导致 cancel_all_orders
   误报"无算法单需要撤销"——上层以为止损撤干净，实际可能残留活止损单。现全部失败时向上抛出，
   调用方既有的降级路径（algo_ok=False / 重试确认）即可生效。
3. **check_and_execute_trades 单品种异常隔离（中）**：原先整轮共用一个 try，任一品种异常会中断当轮
   其余品种的止损推进/平仓检查。现 per-symbol try/except；有失败品种时当日不标记完成，
   由 +1 分钟重试调度整轮重跑（开仓有持仓检查、止损更新有同价跳过、平仓有交易所端张数上限，幂等安全）。
4. **instant_open / close_position 路由加交易锁（中）**：原先与 08:00 日检、盘中巡检无互斥，
   存在并发双开/双平窗口。现非阻塞获取 system._trade_lock，拿不到返回 409。
5. **清理**：equity_tracker 移除上轮口径重构后遗留的死字段 current_drawdown_start；
   api_server 过时注释更正 + Flask>=2.2 的 ensure_ascii 兼容写法。

新增测试：PerSymbolIsolationTest 2 用例（单品种异常不阻断其余品种 / 全成功才标记当日完成）。
本机 32 用例全绿（discover 与倒序双跑）。

**复审重点**：上述 1/2 在 okx_api.py，3 在 main.py check_and_execute_trades，4 在 api_server.py。
请确认 3 的重跑幂等论证成立，以及 1 不影响你环境中 ccxt fetch_positions 的返回结构假设。

## 对第四轮 3 条交易安全意见的逐条回应

1. **contractSize fail closed（已修）**：`_get_contract_size` 获取失败或字段缺失时抛 `ContractSizeUnavailable`，
   不再默认 1.0；换算入口 `_coin_to_contracts` / `round_quantity` 向上传播，开仓/平仓/止损换算一律放弃本次交易。
   日检场景由 per-symbol 隔离捕获并进失败品种钉钉汇总；即时开仓路由返回 500 与错误信息。
2. **撤算法止损可验证（已修）**：`_cancel_algo_order` 改为验证式——尝试所有取消参数组合后，
   **以「算法单列表查无此 id」为成功标准**；OrderNotFound 不再直接当成功（某参数组合把 id 当普通单查也报它）。
   `cancel_order` 的普通单 OrderNotFound 分支同样改为列表裁决。
3. **撤不干净则阻断（已修）**：新增 `main._cancel_stop_order_confirmed` 统一封装四处调用
   （止损更新 / 信号平仓 / 双均线翻转 / 开仓回滚）+ api_server 手动平仓路由。不可确认时：
   - `trade_state` 持久化「止损残留」标记（重启不丢），严重告警；
   - 止损更新：**不创建新止损**，本地保持旧止损记录（与交易所现存挂单对应）；
   - 翻转：平仓记账完成但**不反手开新仓**；
   - `_execute_open` 开头检查残留标记，**阻断该品种一切新开仓**；
   - 每日检查前 `_retry_clear_stop_residues` 自动重试清理（只对已无持仓品种盲撤，有持仓品种的"残留"
     即本地记录的现行止损，交由止损更新流程处理），确认清理后解除阻断并通知恢复。
4. **止损创建超时确认严格匹配（已修）**：`_algo_order_matches` 要求 方向+触发价+张数 全部吻合
   （触发价兼容 stopLossPrice/triggerPrice/info.slTriggerPx 等字段，相对容差 1e-6；任何字段读不到一律不匹配，
   宁可重试创建也不误认残留旧单）。
5. **单测（已补）**：本机新增 `test_okx_adapter_safety.py` 14 用例（面值 fail-closed 4 / 验证式撤单 4 /
   严格匹配 6，桩 ccxt 可本机跑）+ 残留阻断 2 用例（含持久化重启）。`tests/test_trading_logic_unittest.py`
   新增：撤旧失败不更新止损、平仓撤失败仍记账并标记、翻转撤失败不反手；并把既有断言对齐现行缓冲通知行为
   （此前部分断言仍按旧的直发通知写，在依赖环境本来就会挂）。
   本机 48 用例 discover + 倒序全绿。

## 终轮主动增强（Fable 5 提出，非审查意见驱动）

**止损自愈巡检**：此前体系对「撤止损」有验证式确认与残留阻断，但对「止损存在」本身没有守护——

## 资金变动同步的时效性修复（用户反馈驱动）

**问题（用户准确回忆的"时效性很短"）**：同步以「最近一次已记录的指数值」为锚，而权益
采样每 5 分钟一次——入金到账后若采样先跑，会用旧除数把入金记成一根"盈利"暴涨点；
此时再点同步，锚恰好取到被污染的值，入金被永久算进指数。安全窗口只有到账后的
第一个采样周期（<5 分钟）。

**修复**：`equity_sync(flow_amount=None)` 支持净变动金额（入金为正/出金为负）——
提供时锚点按「(当前权益 − 净流入) ÷ 旧除数」精确反推，点击时机不再影响正确性；
反推出的变动前权益 ≤0 时拒绝（防正负号/数量级填错）。留空维持旧行为（时效窗口内
点击仍正确）。前端资金管理卡新增金额输入框与两种锚定方式的说明，确认框显示将采用
的锚定方式。API /api/equity_sync 接收可选 flow_amount，钉钉通知带锚定方式。

**顺带修复既有 bug**：全新系统（initial_equity 为 None，从未跑过统计刷新）时点同步，
原实现会在日志 :.2f 格式化处 TypeError → 路由 500。现 or 0 兜底。

新增 3 用例（污染 tick 后精确校正 / 不合理金额拒绝 / 留空旧行为）。
本机 122 用例 + 依赖版 52 用例全绿。
